State-driven options

Instead of a neutral settings form, the options page becomes a recovery and readiness surface. It explains what is broken, what is optional, and what the user should do next.

Why this differs
The popup is not one layout anymore. It is four tailored layouts with different CTAs.
Best fit
Users who often open the extension in different places and need immediate next-step clarity.
Setup section API key card with inline completion state, health check result, and “test connection” action.
Ready section Default model, fallback plan, and note about where popup surfaces detections.
Recovery section Last error, permission notes, and quick path to clear bad configuration.
