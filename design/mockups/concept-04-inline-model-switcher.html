Options shrink to advanced setup

Once model selection moves into the popup, the options page no longer needs to carry the main day-to-day workflow. It becomes a quieter place for secrets, custom inventory, and defaults.

Popup now owns

Model switching, recent model memory, quick testing, and current run context.

Options keeps

API key entry, model inventory management, fallback defaults, and advanced explanations.

Advanced defaults
Set preferred startup model and failure strategy.
Custom model inventory
Add and remove vendor/model ids without crowding the popup.

Options structure

Top section: secure configuration. Middle section: advanced model catalog. Bottom section: fallback behavior and debugging. The main model selector is intentionally absent because the popup already owns that job.
